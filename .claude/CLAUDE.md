# CLAUDE.md

本文件为 Claude Code (claude.ai/code) 提供在当前仓库中工作的指导。

## 项目概述

这是一个 **Obsidian 个人知识库**（非软件开发项目），包含用于工作日志、学习笔记和项目文档的 Markdown 文件。

**使用方法**：请使用 Obsidian 应用程序打开此文件夹来查看和编辑笔记。

## Obsidian 技能

已安装以下 Claude Code 技能用于操作此知识库：

| 技能 | 用途 |
|------|------|
| `obsidian-markdown` | 创建/编辑 Obsidian Markdown 文件（支持 wikilinks、callouts、frontmatter 等 Obsidian 语法） |
| `obsidian-bases` | 创建/编辑 Obsidian Bases 数据库文件（.base） |
| `obsidian-cli` | 与 Obsidian vault 交互（读取、创建、搜索笔记等） |

## 使用规范

1. 所有内容均为 Markdown 格式（.md 文件）
2. 笔记内链使用 Obsidian wikilinks：`[[笔记名称]]`
3. 标签格式：`#项目/郑州全搜索`、`#领域/编程/PHP`
4. 日记格式：`日记/YYYY-MM-DD.md`
5. 使用中文标点符号（全角符号）

## 注意事项

- 这不是代码项目，没有构建、lint 或测试命令
- 不要尝试运行笔记中找到的代码或脚本，除非用户明确要求
- 优先使用 obsidian-markdown 技能处理 .md 文件

## 文档排版优化

当用户要求优化笔记文档排版时，应遵循以下原则：

### 优化目标
- 移除冗余图片链接
- 删除空行和重复内容
- 提取核心内容为表格或列表
- 统一添加/修复 frontmatter
- 精简代码示例

### 优化流程
1. 使用 `Skill(obsidian-markdown)` 技能处理 .md 文件
2. 读取文件内容，分析结构
3. 提取核心内容，去除冗余
4. 使用表格或列表重构内容
5. 确保 frontmatter 格式规范

### 注意事项
- 不处理超大文件（>256KB）
- 保留原始链接（外部链接、wikilinks）
- 敏感信息（如 Token、密码）需移除或脱敏
